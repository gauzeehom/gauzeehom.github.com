GAE 学习笔记（以部署 micolog 为例）

发表于 2011-07-12   |   分类于 学习记录

一、准备工作：
1.注册应用程序：点这里创建、管理你的 App Engine 网络应用程序（需要 Google 账户）。登录后，点击“创建应用程序”按钮，按照说明注册一个应用程序 ID（application-id），即该应用程序唯一的名称。这里选择使用免费的 appspot.com 域名，那么稍后我们部署的应用程序的完整网址就应该为 http://application-id.appspot.com。

2.下载 Micolog：点这里下载。

二、环境搭建：
1.Python 安装：请到这里（要翻墙，有木有？）下载 Python 2.5.4（官网上要求的是2.5的，所以就没去试其他版本的了）在 Windows 下的安装包进行安装。

2.Google App Engine SDK for Python 安装：请到这里下载最新版的 Windows 安装包进行安装。

三、开始部署：
1.解压下载来的 Micolog，到根目录下找到文件 app.yaml，打开将 application 参数设置为刚才注册的应用程序 ID。

2.运行 cmd，将路径转到文件夹 micolog 所在的目录，然后输入命令：appcfg.py update micolog，回车。按提示输入 Google 用户名和密码。如果不出意外的话，恭喜你，部署工作完成了。

3.打开浏览器，输入网址 http://application-id.appspot.com，好了，这就是你自己的博客了。

四、问题总结：
1.初次输入网址 http://application-id.appspot.com 浏览时，可能会有一个错误代码为 500 的错误提示，别急，过一会再试试也许就可以了。

2.如果你用的是 Google 的浏览器的话，那么恭喜你，当你点击 “(RSS)” 连接时，浏览器将直接为你打开 views 文件夹中的 rss.xml 文件。（提示：换 IE 或火狐试试）

3.修改源代码后，中文出现乱码。解决方法：以“UTF-8 无 BOM 格式”保存所修改的文件。（例如，Notepad++ 编辑器的“以 UTF-8 无 BOM 格式编码”）

4.插入图片问题：首先在“控制面板——>文件”里上传你的图片，接着就可以在“插入/编辑图像”里选择所上传的图片了（点击“图像地址”旁边的“浏览”按钮），选好图片后，看到图像地址为“http://application- id.appspot.com/media/一串字符/图片名称”，点击“插入”后，再看图像地址会变成“../media/一串字符/图片名称”，这时如果发布文章，在文章中是看不到图片的。解决方法是：点击“Visual”旁边的“HTML”找到插入图片的代码，类似“<img src=”../media/一串字符/图片名称”/>”，将其修改为类似“<img src=”http://application-id.appspot.com/media/一串字符/图片名称"/>”的代码，点击“保存”。 再去看看，应该就可以看到图片了。
